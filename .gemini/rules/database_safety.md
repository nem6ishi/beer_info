# データベース安全規約 (DB Safety Rules)
1. 本番データベースのレコードを削除（DELETE）または更新（UPDATE）する際は、必ず事前に SELECT 文で「対象件数」と「データの中身のサンプル」をコンソールに出力し、ユーザーの意図に沿っているか（間違って全件ヒットしていないか）を確認すること。
2. ターミナルから `python -c` などのワンライナーを使って本番DBを一括削除・更新することを絶対に禁止する。データのクリーンアップなどが必要な場合は、必ず `cli.py` に実装された安全なコマンド（`--dry-run` がデフォルトのもの）を使用すること。
3. 大きな破壊的変更を加える前には、可能な限り事前にバックアップ処理（`cli.py sync` や JSONへのダンプ等）を実行しておくことを検討すること。
</RULE[database_safety]>
